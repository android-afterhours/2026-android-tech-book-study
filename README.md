## 📂 GitHub 구성

* **Code**: 예제 코드
* **Issue**: 리딩자 ↔ 스터디원 사이에 질문
* **Wiki**: Issues + Code 기반으로 해당 챕터 요약 작성

## 🔄 운영 방식

- **주기**: 1 ~ 2주 단위로 책을 읽는다
- **역할**: 한 사람씩 돌아가면서 리딩자를 맡는다
- **질문 등록**: 스터디원은 토요일까지 이슈에 질문 업로드
  - ex. 스레드 블로킹에 대한 개념 및 예제 코드가 있었으면 좋겠어요
  - ex. joinAll()을 구체적으로 어떤 상황에서 쓰면 좋을지, 구체적인 예시가 있었으면 좋겠어요!
- **답변 작성**: 리딩자는 목요일까지 이슈에 답변 달기
  - 스터디원은 답변 미리 확인 후 추가 질문 가능
- **스터디 진행**: 목요일 8시 30분 ~ 9시 30분 디스코드
  - 이슈 답변을 함께 읽으며 이해되면 **close**, 그렇지 않으면 **stuck** 처리
  - Stuck 항목은 추가 답변 후 주기적으로 모아서 close 여부 확인
- **정리**: 스터디 종료 후, 리딩자는 이슈 답변을 포함해 위키에 챕터 요약 작성

### 🕐 일정 예외 처리
* 수요일로 변경 가능
  * 당일 불참 통보 → 커피 쏘기 ☕
  * 화요일까지 통보 → OK ✅

## 💡 참고 Tip: 이슈 질문 및 답변 방법

### 질문 예시
- 처음 보는 용어에 대한 설명 요구
- 글로만 적혀있는 기술에 대한 사용법 요구
- 예제 코드에 대한 설명
- 문단 문맥 이해
- 코드에 대한 분석 요구 (주로 JDK 구현 방식에 대한 상세 설명)
- 관련 기술 사용 예제 요구

### 답변 방법
- 구글링을 통한 기술 블로그 참고
- 공식 문서 확인
- 실제로 코드를 작성하여 해결
- JDK 내부 구현을 확인하여 해결
- AI의 도움을 받아도 좋으나, **증명할 수 있는 자료 포함**
