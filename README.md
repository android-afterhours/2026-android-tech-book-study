# 2026-android-tech-book-study
> 안드로이드 개발 관련 서적 읽고 서로 질문하면서 성장하는 스터디

## 운영 방식

- 1 ~ 2주 단위로 책을 읽는다. 
- 한 사람씩 돌아가면서 리딩자를 맡는다. 그 외 나머지 멤버는 이슈에 이해가 되지 않는 문장이나 기록으로 남겨두고 싶은 예제 등을 올린다. 
    - ex. 스레드 블로킹에 대한 개념 및 예제 코드가 있었으면 좋겠어요.
    - ex. joinAll()을 구체적으로 어떤 상황에서 쓰면 좋을지, 구체적인 예시가 있었으면 좋겠어요! 
- 리딩자는 스터디 전까지 이슈에 답변을 남긴다. 
- 스터디 당일에는 같이 이슈에 남긴 답변을 읽으면서 모두 동의하면 close, 그렇지 않은 경우 stuck 처리 한다. 
- 스터디가 끝난 후, 리딩자는 이슈에 남긴 답변을 포함해서 해당 챕터에 대한 내용을 올린다
